# 项目结构说明

## 目录分层
- `app/`: Next.js App Router 页面与 API 路由
- `components/`: 可复用 UI 组件与交互组件
- `lib/`: 业务逻辑、权限规则、数据访问辅助、共享常量
- `prisma/`: 数据模型、迁移与种子
- `public/`: 静态资源
- `docs/`: 设计与实现文档

## 当前约定
- 页面展示逻辑放在 `app/**/page.tsx`
- 管理端 API 放在 `app/api/admin/**`
- 会员端 API 放在 `app/api/member/**`
- 跨页面复用的配置（如栏目树）放在 `lib/`

## 建议后续持续执行
- 新增业务枚举、栏目配置、权限映射时优先放 `lib/`，避免页面内重复定义
- 页面中仅保留渲染与事件处理，不再散落硬编码栏目字符串
- 临时调试文件统一使用 `.tmp-*` 命名（已在 `.gitignore` 忽略）
