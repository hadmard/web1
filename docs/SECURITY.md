# 安全防护实施清单（3.0）

依据《中华整木网 3.0 改版执行指令》第八条，本清单为强制检查项。

## 已实施 / 部署层

- **HTTPS 强制跳转**：由部署环境（Vercel / Nginx 等）配置，生产环境必须启用 HTTPS。
- **防 SQL 注入**：使用 Prisma ORM，所有查询参数化，无裸 SQL 拼接。
- **权限分级控制**：后台区分 SUPER_ADMIN（主账号）与 ADMIN（子管理员），子管理员无添加账号权限。

## 需在部署与运维中落实

- **后台登录二次验证**：可选集成 TOTP 或邮件验证码，建议在后续迭代中接入。
- **上传文件安全检测**：若开放上传，需校验文件类型、大小，并存储至安全路径，避免执行权限。
- **定期数据库备份**：由运维或托管方配置自动备份（如 SQLite 文件备份、PostgreSQL 快照）。
- **异常访问监控**：通过服务器日志或 APM 监控异常请求与登录失败率。
- **日志追踪系统**：关键操作（登录、内容发布、权限变更）建议落库或写入日志，便于审计。

## 前端与内容安全

- **防 XSS**：用户生成内容输出须转义；若使用 `dangerouslySetInnerHTML`（如资讯正文），应仅对可信内容或经 sanitize 的内容使用，或改用安全富文本方案。
- **CSP**：可在 `next.config.mjs` 或响应头中配置 Content-Security-Policy，限制脚本与资源来源。
