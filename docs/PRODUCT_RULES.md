# 中华整木网 · 产品规则

## 定位与设计原则

- **行业**：整木定制行业垂直门户。
- **定位**：整木定制行业知识基础设施网站。
- **设计**：符合 SEO 抓取与当下流行的 GEO（AI 检索引用），详见 [SEO 与 AI 检索引用规则](./SEO_AND_AI_RULES.md)。

---

## 账号与权限

### 主账号（SUPER_ADMIN）

- **职责**：站点级配置与人员管理。
- **能力**：
  - **修改大类、小类名称**：在后台修改各栏目（大类）及其下小类的显示名称，供前台与 SEO 使用。
  - **栏目定义、版本、FAQ、相关词条**：编辑各栏目的定义说明（首屏）、版本标签/版本年、常见问题列表、底部相关词条（词条 slug），以及小类的分组标签（groupLabel），详见 [栏目体系升级版](./COLUMN_STRUCTURE.md)。
  - **分配分管理员**：创建分管理员账号（邮箱 + 密码），用于日常内容管理维护。
- **入口**：登录后进入「会员中心」，可见「主账号后台」；或直接访问 `/membership/admin`（需主账号权限）。

### 分管理员（ADMIN）

- **职责**：日常内容管理维护。
- **能力**：词条、标准、品牌、数据、资讯等内容的新增、编辑、删除等（具体能力由后台与 API 实现）。
- **账号**：由主账号在「主账号后台」中创建，无需自行注册。

### 普通会员（MEMBER 或未设 role）

- 仅会员中心、浏览等，无后台管理权限。

## 大类与小类

- **大类**：按用户路径（了解行业→了解标准→对比品牌→建立信任→产生联系）顺序为：整木资讯、整木词库、整木标准、整木市场、整木数据、整木评选、整木图库、会员系统，对应独立首页。详见 [栏目体系升级版](./COLUMN_STRUCTURE.md)。
- **小类**：每个大类下的子栏目（含分组展示，如整木资讯下「行业活动」分组：展会报道、论坛纪要、协会活动），小类下为具体内容。
- **名称与元数据可配置**：大类标题、描述、定义说明、版本、FAQ、相关词条及小类标签/分组均存储在数据库中，**主账号可在后台修改**，修改后全站即时生效。
- **数据来源**：`Category` / `SubCategory` / `CategoryFaq` 表，seed 时从 `lib/site-structure.ts` 同步初始结构；之后以数据库为准，`lib/categories.ts` 的 `getCategories()`、`getCategoryWithMetaByHref()` 供前台读取。

---

## 实施与扩展

- 新增大类/小类：主账号后台可考虑支持“新增栏目”（当前为修改已有名称）；或由开发在 DB 中新增记录并重跑/扩展 seed。
- 分管理员的内容权限：在内容相关 API（词条、标准、品牌等）中校验 `session.role === "ADMIN" || session.role === "SUPER_ADMIN"`，即可实现「分管理员做日常内容维护」。
