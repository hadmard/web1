# 中华整木网 3.0 改版执行指令（最终工程版）

本文档为结构级升级的强制检查清单，所有迭代与 PR 须引用并遵守。

---

## 一、改版性质说明

本次改版为**结构级升级**，不是视觉升级。

### 目标

1. 强化 GEO 引用能力（结构化 + 规范化）
2. 强化会员自主维护能力
3. 强化栏目之间的结构关联
4. 强化自然成交路径
5. 将网站升级为**整木定制行业知识基础设施平台**

### 禁止

- 仅优化 UI
- 增加广告系统
- 增加论坛系统
- 做内容堆积型门户

---

## 二、全站统一结构规范（强制执行）

### 1. 栏目首页统一结构

每个栏目首页必须包含：

- 栏目定义说明（100–200 字，规范化表达）
- 当前版本号（例如：整木标准 2026 版）
- 最近更新时间
- FAQ 问答模块（不少于 3 条）
- 相关栏目推荐区块
- 标签入口区

栏目定义必须采用行业规范表达，便于 AI 抓取。

### 2. 所有内容页面统一结构

所有内容页必须采用如下结构：

1. 概念定义（100 字以内摘要）
2. 正文说明
3. 适用场景 / 结构说明
4. FAQ 问答
5. 相关标准
6. 相关品牌
7. 相关案例
8. 更新时间
9. 版本号

**禁止无结构文章发布。**

### 3. 全站强制标签系统

所有内容必须绑定以下标签类型：

- 行业标签
- 技术标签
- 风格标签
- 区域标签
- 等级标签

标签页必须：可访问、可分页、可被搜索引擎抓取。

---

## 三、GEO 技术实现要求

全站必须支持：

- JSON-LD 输出
- FAQ Schema
- Organization Schema
- Dataset Schema
- Article Schema
- 静态 URL 结构
- 面包屑导航
- 自动生成 sitemap.xml
- robots.txt 管理

---

## 四、安全防护要求（必须执行）

必须包含：

- HTTPS 强制跳转
- 后台登录二次验证
- 权限分级控制
- 防 SQL 注入
- 防 XSS 攻击
- 上传文件安全检测
- 定期数据库备份
- 异常访问监控
- 日志追踪系统

---

## 五、最终系统目标

中华整木网 3.0 必须成为：

- 行业解释系统
- 行业标准系统
- 行业对比系统
- 行业数据系统
- 行业信任系统
- 企业共建系统

**最终定位：整木定制行业知识基础设施平台。**
